# Как собрать и раздать лаунчер

---

## Что должно быть на твоём компе (один раз)

### 1. Node.js
Скачай с https://nodejs.org — берёшь версию **LTS** (левая кнопка).
Установи с галочкой "Add to PATH". После установки проверь в cmd:
```
node -v
npm -v
```
Должны вывести версии, например `v20.11.0` и `10.2.4`.

### 2. Git (опционально, но удобно)
https://git-scm.com — нужен только если хочешь клонировать репо командой.

### 3. Python (только если используешь auth-server)
Тут ты его убрал, так что Python не нужен.

---

## Настройка перед первой сборкой

Открой `src/config.js` и заполни:

```js
SERVER_IP:      'yourserver.playit.gg',  // адрес от Playit.gg
MODS_OWNER:     'твой_github_юзернейм',
MODS_REPO:      'название_репо_с_модами',
LAUNCHER_OWNER: 'твой_github_юзернейм',
LAUNCHER_REPO:  'название_репо_лаунчера',
```

В `package.json` тоже замени в блоке `"publish"`:
```json
"owner": "твой_github_юзернейм",
"repo":  "название_репо_лаунчера"
```

## Установка зависимостей (один раз)

Открой cmd/PowerShell в папке проекта:
```
npm install
```
Скачает Electron и electron-builder (~300 МБ, один раз).

---

## Запуск для разработки / проверки

```
npm start
```
Откроется лаунчер. Можно тестировать без сборки.

---

## Сборка EXE для Windows

```
npm run build:win
```

Результат появится в папке `dist/`:
```
dist/
├── YourServer Launcher Setup 1.0.0.exe   ← это и раздаёшь игрокам
└── win-unpacked/                          ← портативная версия
```

Установщик (`Setup .exe`) — тихая установка одним кликом, без лишних вопросов.
Ставит лаунчер в `C:\Users\...\AppData\Local\YourServer Launcher\`.

### Если хочешь иконку
Положи файл `assets/icon.ico` (256×256 px).
Сделать `.ico` из PNG можно на https://convertio.co/png-ico/

## Выпуск обновления лаунчера через GitHub Releases

Когда поменял что-то в лаунчере:

1. Увеличь версию в `package.json`:
   ```json
   "version": "1.0.1"
   ```

2. Собери:
   ```
   npm run build:win
   ```

3. Зайди в GitHub → твой репо → **Releases** → **Draft a new release**

4. Тег: `v1.0.1` (обязательно с `v` в начале)

5. Прикрепи файл `dist/YourServer Launcher Setup 1.0.1.exe`

6. Нажми **Publish release**

При следующем запуске старого лаунчера появится жёлтый баннер с предложением обновиться.

## Репозиторий с модами (структура)

Создай отдельный GitHub-репозиторий (может быть приватным, но тогда Raw-ссылки не работают — делай публичным).

```
твой-mods-репо/
├── manifest.json
├── NEWS.json
├── mods/
│   ├── journeymap-1.20.1-5.9.18.jar
│   ├── jei-1.20.1-forge.jar
│   └── ...
└── config/
    └── forge.cfg
```

**manifest.json** — список файлов и версия:
```json
{
  "version": "1.0.0",
  "mcVersion": "1.20.1",
  "files": [
    { "path": "mods/journeymap-1.20.1.jar" },
    { "path": "mods/jei-1.20.1.jar" },
    { "path": "config/forge.cfg" }
  ]
}
```

Чтобы обновить моды — добавь/замени файлы и **увеличь `"version"`**.
Лаунчер увидит разницу при следующем запуске и скачает изменения.

**NEWS.json** — новости сервера:
```json
[
  {
    "id": 1,
    "date": "2024-01-15",
    "title": "Сервер открыт!",
    "body": "Добро пожаловать! Подключайся через лаунчер."
  },
  {
    "id": 2,
    "date": "2024-01-20",
    "title": "Обновление модов 1.0.1",
    "body": "Добавлен JourneyMap. Обнови клиент через лаунчер."
  }
]
```

---

## Что получают игроки

1. Скачивают `YourServer Launcher Setup x.x.x.exe`
2. Запускают — тихая установка за 5 секунд
3. Вводят никнейм → лаунчер качает моды → жмут Играть

Моды скачиваются в `C:\Users\...\AppData\Roaming\.mc-launcher\client\`

## Краткая шпаргалка

| Задача | Команда |
|--------|---------|
| Проверить как выглядит | `npm start` |
| Собрать EXE | `npm run build:win` |

| Выпустить обновление лаунчера | Поменять `version` в `package.json` → собрать → GitHub Release |
